# Eco-evolutionary dynamics and environmental detoxification shape bacterial community response to ampicillin perturbation

This repository contains downstream analysis code and processed data tables to reproduce the analyses and figures for the manuscript:

The repository includes:
- processed/curated downstream data tables (`data/`)
- analysis notebooks and shared settings (`notebooks/`)
- directory (`figs/`) for PDF files for figures generated by notebooks

---

## Requirements

- **R 4.2.2**
- A working Jupyter environment capable of running R notebooks (e.g. JupyterLab with an R kernel), or an R workflow that can execute the same code contained in the notebooks.

---

## Repository structure

- `data/`  
  Downstream tables used for all analyses and figure generation.

- `notebooks/`  
  Notebooks and shared settings used to reproduce analyses and figures:
  - `01_Fig2_related_analyses.ipynb`
  - `02_Fig3_related_analyses.ipynb`
  - `03_Fig4_related_analyses.ipynb`
  - `04_Fig5.ipynb`
  - `05_mutation_analysis.ipynb`
  - `06_CNV_analysis_READY.ipynb`
  - `07_Extended_Data.ipynb`
  - `project_settings.R`

- `figs/`  
  Figure PDFs produced by the analysis notebooks:
  - `Fig2.pdf`, `Fig3.pdf`, `Fig4.pdf`, `Fig6.pdf`
  - `ED_Fig1.pdf` … `ED_Fig6.pdf`

---

## Running the analyses

### Working directory and paths

The notebooks read inputs from `../data/` and write outputs to `../figs/`.  
For this reason, run notebooks with the working directory set to `notebooks/` (or ensure relative paths resolve identically).

### Reproducing figures

1. Open a notebook under `notebooks/`
2. Run all cells
3. Confirm updated outputs in `figs/`

A typical figure notebook writes the corresponding PDF to `figs/` (e.g. `01_Fig2_related_analyses.ipynb` writes `figs/Fig2.pdf`).

To force regeneration of all figure PDFs:
